Title - Decorator pattern  (structural)

Motivation to refactor - If we need to add new functionality to an existing object without altering its structure, we can use the decorator pattern. This allows putting new functionalities into a different class with unique features of it and adds those unique features to an existing object. 

Pattern/refactoring method - By using the decorator design pattern, we can add new functionalities into an existing object without affecting its structure.
